Features

A feature-complete 1.20.1 survival server.

Built as a Cargo workspace of focused crates that snap together — protocol, world, mods, server.

🎮

Real client compatibility

A vanilla 1.20.1 client connects, logs in, and walks around a lit, generated world. Server List Ping shows a live MOTD and player count.

🔐

Login & encryption

Offline-mode UUIDs, packet compression, and optional online-mode encryption with RSA key exchange and an AES-CFB8 stream.

🌍

World generation & lighting

Superflat or smooth value-noise terrain with paletted chunk sections and heightmaps, plus real skylight and block-light flood-fill.

🧱

Full block registry

All 1003 of 1.20.1's blocks resolvable by name for /setblock, /give, and mods, with correct light opacity and emission.

👾

Mobs, AI & combat

~80 living entity types spawn, wander with gravity, and hostiles chase, melee, shoot arrows or explode. Health/food HUD, knockback, death and respawn.

🎒

Items, survival & crafting

A real 46-slot inventory, drops and pickup, hunger drain, XP, a recipe engine for /craft and the grid, plus working furnaces.

🔴

World simulation & redstone

Crops grow, trees and grass spread, water and lava flow, fire burns out — and full redstone: wire, repeaters, comparators, observers, and pistons.

💾

Persistence backends

Choose delta (edits over the generator) or region (Anvil-style columns). Chests, signs, players, mobs, vehicles, and the clock all persist.

📜

JavaScript mods

Drop a .js file in mods/ — a sandboxed QuickJS runtime with an event and action API to react to the world and add commands.

🧑‍🌾

AI villagers (experimental)

Opt in to give villagers LLM-driven personalities via Ollama, OpenAI, or Claude — right-click a villager and it chats back in character for its profession.

🛰️

Control API & Atlas panel

HTTP + WebSocket endpoints for status, players, broadcast, kick, and set-block — driven by a live Atlas web admin dashboard.

🔁

Multi-version

A bidirectional translation engine serves nine protocol versions across eleven MC releases, 1.18 through 1.20.4, from one native 763 core.

Architecture

Three layers, one Cargo workspace.

  • ✦Engine — Rust (Tokio) — the server itself: protocol, world, chunks, players, and the game loop. Focused crates that snap together.
  • ✦Mods — JavaScript via QuickJS — drop-in .js mods (on rquickjs) that react to events and act on the world.
  • ✦Control panel — TypeScript + Bun on Atlas — a live web admin UI that drives the engine's control API.
  • ✦Configuration — everything lives in cubeplane.toml and is optional: port, gamemode, generator, seed, mods, and the control port.
  • ✦No external noise crates — terrain, lighting, and the registry are all built in-house from authoritative Minecraft data.

Get started

Running in a few minutes.

Requires Rust (Cargo) for the engine and Bun for the optional admin panel. Connect with Minecraft Java Edition 1.20.1.

terminal 
# 1. run the engine (reads ./cubeplane.toml)
cargo run --release
# → cubeplane listening on 0.0.0.0:25565 — Minecraft 1.20.1 (protocol 763)
# → control API listening on http://127.0.0.1:8080

# 2. connect from Minecraft Java Edition 1.20.1 → server: localhost

# 3. launch the Atlas admin panel
cd admin
bun install   # pulls Atlas from GitHub
bun run dev   # → http://localhost:3000

Write a mod by dropping a .js file in mods/ and restarting:

mods/welcome.js 
cubeplane.on("player_join", (e) => {
  cubeplane.broadcast(e.player + " joined the cubeplane!");
});

cubeplane.command("tower", (ctx) => {
  for (let i = 0; i < 10; i++) cubeplane.setBlock(0, 4 + i, 0, "stone");
});
